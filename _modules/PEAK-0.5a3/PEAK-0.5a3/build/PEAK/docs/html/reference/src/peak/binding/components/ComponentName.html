Path between components

Component Path Syntax

Paths are "/" separated attribute names. Path segments of "." and ".." mean the same as they do in URLs. A leading "/" (or a compound name beginning with an empty path segment), will be treated as an "absolute path" relative to the component's root component.

Paths beginning with anything other than "/", "./", or "../" are acquired, which means that the first path segment will be looked up using acquireComponent() before processing the rest of the path. (See acquireComponent() for more details.) If you do not want a name to be acquired, simply prefix it with ./ so it is relative to the starting object.

All path segments after the first are interpreted as attribute names to be looked up, beginning at the component referenced by the first path segment. '. and ..' are interpreted the same as for the first path segment.
