Class interface for creating ICmdLineApp components

A command-line app object is created with keyword arguments for stdin, stdout, stderr, environ, and argv. It is free to use default values for items not supplied, but it must not bypass or override any values which are supplied. E.g. it should never write to sys.stdin. The purpose of this encapsulation is to allow application objects to be composed by other application objects, and to also allow "server" invocations of applications, as is needed for protocols like FastCGI and ReadyExec.
